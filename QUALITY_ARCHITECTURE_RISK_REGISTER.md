# Cosmic Almanac 质量与架构风险清单（轻量保守版）

更新日期：2026-04-14

## 范围与约束

- 目标：以最小改造成本交付可执行风险清单，不引入构建工具或后端。
- 覆盖范围：运行可靠性、架构一致性、可维护性、PWA 离线行为、文档一致性。
- 约束：不变更用户可见交互语义（生日输入、语言切换、主题切换、日期切换、详情弹窗）。

## 风险分级清单

## `P0`（立即处理）

### `P0-1` Service Worker 子路径缓存失效

- 影响面：GitHub Pages 项目子路径部署（如 `/StarGate/`）可能无法安装 SW，导致离线不可用与缓存策略失效。
- 触发条件：`sw.js` 使用站点根路径（`/index.html`、`/app.js` 等）执行 `cache.addAll()`。
- 建议修复：缓存资源改为基于 SW scope 的相对路径（`./index.html` 等），并在升级时更新缓存版本号。
- 验收方式：
  1. DevTools `Application -> Service Workers` 可见已安装并激活。
  2. `Cache Storage` 中存在当前缓存版本并包含应用壳资源。
  3. 断网刷新可命中缓存并正常加载主页。
- 本轮状态：`已落地`（`sw.js` 已切换为 scope 相对路径，缓存版本升级为 `cosmic-v2`）。

## `P1`（短期处理）

### `P1-1` 文档与代码结构分叉

- 影响面：新贡献者会按错误结构维护（误以为 `content.js` 单文件、样式写在 `index.html`），增加回归风险。
- 触发条件：README/Wiki 长期保留旧架构描述。
- 建议修复：统一文档中的“文件结构、部署清单、架构图”到实际加载链：`index.html + app.css + app.js + content.en.js + content.zh.js + sw.js`。
- 验收方式：
  1. README 与 Wiki 中不再要求 `content.js`。
  2. 文件清单和部署清单与仓库现状一致。
- 本轮状态：`已落地`（README、Wiki、内容报告已同步）。

### `P1-2` SW 缓存升级流程依赖人工约定

- 影响面：发布后用户可能长期命中旧缓存，导致“修复已上线但前端仍旧”的陈旧问题。
- 触发条件：资源变更但未同步提升 `CACHE` 版本号。
- 建议修复：保留手动版本策略，并在发布流程加入“改动应用壳资源时必须 bump CACHE”的检查项。
- 验收方式：
  1. 发布 checklist 包含缓存版本检查。
  2. 连续两次发布可观察到旧缓存被清理、新缓存生效。
- 本轮状态：`部分落地`（已补充版本号注释并升级到 `cosmic-v2`，发布流程检查项待团队执行）。

## `P2`（中期改进）

### `P2-1` `app.js` 单文件耦合过高

- 影响面：`renderReading` 同时承担模型计算、状态变更、DOM 渲染，后续需求迭代容易产生连带回归。
- 触发条件：继续在单函数叠加业务逻辑或新增卡片/详情类型。
- 建议修复（最小改造，不引入构建工具）：
  1. 先约定内部边界：`计算层（纯函数）` 与 `渲染层（DOM 操作）` 分离。
  2. 首步仅抽离一个 `buildReadingModel(...)` 纯函数，保留现有 UI 行为不变。
  3. 后续按功能块继续拆分 `detail payload 构建` 与 `DOM patch`。
- 验收方式：
  1. `renderReading` 代码长度明显下降且以“读取 model -> 渲染”为主。
  2. 抽离函数不依赖 DOM，可独立验证输入输出。
- 本轮状态：`待执行`（本轮不做大规模重构，保留行为稳定优先）。

## 最小改造执行顺序（本轮）

1. 先修运行可靠性：修 SW 子路径缓存失效（`P0-1`）。
2. 再修一致性：同步 README/Wiki/报告到真实架构（`P1-1`）。
3. 最后控演进风险：记录单文件耦合改造边界和后续步骤（`P2-1`）。

## 内部接口约束（保持现有用户行为）

- 不新增外部 API。
- 保持现有页面交互语义和数据字段不变。
- 仅调整：
  1. 静态资源加载约定（相对 SW scope 路径）。
  2. 文档单一事实来源（README/Wiki 与仓库结构一致）。

## 验证清单（静态项目可执行）

### 功能回归

1. 首次访问：输入生日并进入阅读页。
2. 重置生日：确认弹窗后回到 onboarding。
3. 切换语言/主题：页面文案与主题即时更新。
4. 日期切换：前一天/后一天/回到今天均可工作。
5. 稳定性：同生日同日期结果稳定，跨日期结果变化。

### PWA

1. 首次加载后 SW 成功注册并激活。
2. 断网刷新首页可用。
3. 升级缓存版本后旧缓存被清理。

### 文档一致性

1. README 与 Wiki 文件清单匹配仓库实际文件。
2. 部署说明中的必要文件与当前加载链一致。
