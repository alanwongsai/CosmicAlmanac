# 界面与设计系统

## 设计方向

Cosmic Almanac 的 V1 视觉方向是 liquid glass、星象、每日仪式感和轻量神谕。界面应该像一个精致的小工具，而不是复杂后台或传统占星软件。

## 核心视觉元素

- 深色夜空背景。
- 白天模式的暖色浅背景。
- 星点层和柔和渐变。
- 毛玻璃卡片。
- 柔和内高光。
- 香槟金强调色。
- 圆角卡片和按钮。
- 轻微浮动和淡入动画。

## 页面结构

应用主要有两个页面状态：

### Onboarding

首次进入时显示生日输入页，包括：

- App 图标。
- App 标题。
- 简短说明。
- 生日输入框。
- 开始按钮。

### Reading

输入生日后显示阅读页，包括：

- 日期和今日标题。
- 西方星座和中国生肖标签。
- 今日天象卡片。
- 日期导航。
- 四个评分卡片。
- 今日神谕。
- 宇宙建议。

## 交互设计

大多数卡片都可点击。点击后打开居中详情弹窗，显示更完整的解释。

详情弹窗用于解决两个问题：

- 首页保持紧凑。
- 用户想深入了解时可以获得更多上下文。

今日宜忌弹窗包含 tab 式内容，可以在月相、月入星座和行星日之间切换。

## 语言与主题按钮

早期版本中，语言和日夜按钮使用固定定位，始终停留在屏幕上方。这在长页面中可能遮挡内容。

V1.0.2 已将它们改为页面顶部控制栏：

- 仍然显示在页面顶部左右两侧。
- 向下滚动时会随页面一起离开视口。
- 不再覆盖阅读卡片。

## 字体策略

英文模式使用更接近书籍印刷感的衬线字体栈，让内容读起来更像每日短文或神谕卡片。

中文模式使用系统中文字体栈，保证可读性和移动端显示稳定。

## 响应式策略

V1 主要面向手机竖屏，同时也能在桌面浏览器中居中显示。关键宽度限制包括：

- 主要内容最大宽度约 430px。
- Onboarding 卡片最大宽度约 375px。
- 小屏下今日宜忌布局会从双列变成单列。
- 窄屏下部分卡片 padding 和字号会收紧。

## 设计限制

当前版本的视觉资产较少，主要依靠 CSS 背景、图标 emoji 和现有 PWA 图标。后续如果要增强分享和品牌感，最值得补的是 1200 x 630 社交分享图，以及一组更正式的品牌视觉资产。
