# 开发历程与路线图

## 开发历程概览

Cosmic Almanac 从一个单页星象应用逐步迭代到 V1 完整版本。根据当前项目记录，主要演进路径如下：

1. 建立基础单页应用。
2. 添加西方星座、中国生肖、月相和每日解读。
3. 增加交互卡片和详情弹窗。
4. 补充科学边界和象征性说明。
5. 加入中文 UI 与中文内容。
6. 加入今日天象和今日宜忌系统。
7. 重构 almanac 逻辑并优化小屏布局。
8. 拆分内容文件为 `content.en.js` / `content.zh.js`，并将页面样式与逻辑抽离到 `app.css` / `app.js`。
9. 添加内容数据库报告和代码学习笔记。
10. 删除早期本地分享功能，保持 V1 简洁。
11. 发布 V1，并迭代到 `1.0.2`。
12. 将项目正式命名为 Cosmic Almanac。
13. 补齐链接分享元数据。
14. 修复语言和日夜按钮固定遮挡问题。

## 重要里程碑

### 内容拆分

早期内容和逻辑集中在 `index.html`。后来将结构、样式、逻辑和内容分层为 `index.html`、`app.css`、`app.js`、`content.en.js`、`content.zh.js`，让主程序和内容库分离。

这个调整降低了后续扩展文案、增加语言和调整规则的成本。

### 今日宜忌系统

今日宜忌从简单展示升级为五因素组合系统：

```text
月相 × 月入星座 × 行星日 × 西方出生星座 × 中国生肖
```

这让 V1 拥有 96,768 种组合可能，同时不需要手写同等数量的完整文案。

### 详情弹窗

交互卡片和详情弹窗让首页保持紧凑，同时给愿意深入阅读的用户提供更多解释。

### 双语支持

项目支持 English 和简体中文。语言切换不仅影响界面文字，也影响星座、生肖、月相、行星日、评分和详情文案。

### PWA 和分享包装

项目加入 `manifest.json`、应用图标、Apple touch icon、theme color 和基础 Open Graph / Twitter Card 元数据，使它更适合被安装和分享。

## 当前 V1 后的优先路线图

### P1：分享视觉资产

制作 1200 x 630 的 social preview 图，并添加：

- `og:image`。
- `twitter:image`。
- GitHub 仓库 Social preview。

这会明显改善聊天软件和社交平台中的链接预览效果。

### P2：内容深度扩展

优先扩展评分详情弹窗：

- 从 `维度 × 分数档` 扩展到 `维度 × 分数档 × 多模板`。
- 或为每条评分短句绑定独立扩写。

### P3：更准确的历法边界

后续可以改进：

- 中国生肖按农历新年边界计算。
- 月入星座改用更精确的天文算法或数据表。

### P4：产品化设置

可考虑增加：

- 重置生日入口的更明显 UI。
- 隐私说明页。
- 关于页面。
- 离线缓存 service worker。
- 更正式的品牌图标和分享图。

### P5：架构升级

如果页面继续增长，可以考虑：

- 将 JavaScript 拆成多个模块。
- 将 CSS 拆出单独文件。
- 引入轻量构建工具。
- 用测试覆盖关键计算函数。

## 当前推荐下一步

V1 已经足够完整。最值得做的下一步不是继续堆功能，而是完善对外包装：

- GitHub 仓库 About 描述。
- GitHub Pages 链接确认。
- Social preview 图。
- 真正 GitHub Wiki 页面的同步。
